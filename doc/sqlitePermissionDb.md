# Permission DB based on SQLite3

This database store information about Android permissions,
which have used during Android app parsing.

# Requirements

sqlite3 version 3.30.1

For Linux you can use the following [link].(https://www.sqlite.org/2019/sqlite-tools-linux-x86-3300100.zip)

# Schemes

In this database there are two tables.
Below presented the example of two tables

## Permissions

| Name | Protection |
| ---- | ---------- |
| ACCESS_CALL_AUDIO | 6 |
| ACCESS_CHECKIN_PROPERTIES | 2048 |
| ACCESS_COARSE_LOCATION | 1 |

## Protection levels

The value is based on bit mask

| Value | Name |
| ----- | ---- |
| 1 | Dangerous |
| 2 | Signature |
| 4 | Appop |
| 8 | Privileged |
| 16 | Development |
| 32 | Normal |
| 64 | Instant |
| 128 | Preinstalled |
| 256 | RetailDemo |
| 512 | Installer |
| 1024 | Pre23 |
| 2048 | Unused |
| 4096 | Deprecated |

# SQLite source code

## Create databases
```sqlite
CREATE TABLE IF NOT EXISTS permissions (
  name TEXT PRIMARY KEY,
  protection INTEGER
) WITHOUT ROWID;

CREATE TABLE IF NOT EXISTS protection_levels (
  value INTEGER PRIMARY KEY,
  name TEXT
) WITHOUT ROWID;
```
